## 1. プロジェクト概要

### 1-1. 背景と目的

当院では KING OF TIME（勤怠管理クラウド）に打刻データをCSVインポートする運用を行っている。現在は単一HTMLファイル（Ver 4.0）を各ドクターに配布し、各自の端末で有給・休日出勤・時間変更を入力してCSVをダウンロード、担当者（山本さん）に手動で提出している。

Ver 5.0 では以下の課題を解決する：

- HTMLファイル配布 → URLアクセスに変更し、アップデートを全員に即時反映
- 端末依存のlocalStorage → サーバー保存（Cloudflare KV）に移行し、データの永続性を確保
- 手動CSV提出 → 月末自動送信により、各ドクターの提出忘れ・担当者の催促作業を削減

### 1-2. 対象ユーザー

| ロール | 対象者 | 操作内容 |
|--------|--------|----------|
| ドクター | 生野、岡田、露口、松浦、藤田、加藤、岸元、安達、松本、久保、落窪、山本、珠央 | 自分の勤怠例外データを登録、確定ボタンで送信予約 |
| 事務担当 | 山本さん | 月末に全ドクターのCSVをメールで受信、KING OF TIMEにインポート |
| 管理者 | 生野（院長） | アプリの設定・保守、未確定ドクターへのリマインド確認 |

### 1-3. 技術スタック

| 技術 | 用途 |
|------|------|

### 2-2. メイン画面（既存画面の改修）

Ver 4.0 の画面構成を基本的に維持しつつ、以下を変更する。

**ヘッダー変更点**
- 左側：アプリロゴ + タイトル（変更なし）
- 右側：「◯◯先生」表示 + 「ドクター変更」ボタン（タップで選択画面に戻る）

**STEP 1: 基本情報設定の変更**
- 従業員ID・氏名の入力欄を削除（ドクター選択画面で決定済みのため）
- ドクターID参照リスト（アコーディオン）を削除
- 対象年月・定休日設定・祝日データ確認はそのまま維持

**STEP 2: 例外スケジュールの修正（変更なし）**
- カレンダービュー（クリックで出勤/休み切替）
- お休みリスト（祝日・有給・欠勤・振替休日）
- 休日出勤リスト（振替診療含む）
- 時間変更リスト（遅刻・早退・残業）

**STEP 3: プレビュー＆アクション（変更あり）**
- 「データを作成・プレビュー」ボタン → 集計サマリー + テーブルプレビュー（変更なし）
- 「CSVファイルを保存する」ボタン（既存、ローカルダウンロード用に維持）
- 「確定して送信予約する」ボタン（新規追加） → 後述の確定フロー

### 2-3. 確定フロー（新規追加）

プレビュー画面の下部に「確定して送信予約する」ボタンを追加する。

**ボタンの動作**
1. ドクターがボタンをタップ
2. 確認ダイアログ表示：「2026年4月の勤怠データを確定します。月末に山本さんへ自動送信されます。よろしいですか？」
3. 「確定する」を押すと、KVに確定ステータスを保存
4. ボタンが「確定済み（4/8 確定）」表示に変わり、再押下不可になる
5. 「確定を取り消す」リンクを小さく表示（取り消すと再編集可能に戻る）

**確定ステータスのKV保存**
- キー：`confirmed:{empId}:{yyyy-MM}`（例：`confirmed:1030:2026-04`）
- 値：`{ confirmed: true, confirmedAt: "2026-04-08T15:30:00+09:00", empName: "生野智也" }`

**確定後の挙動**
- 確定済みの月のデータは編集不可（カレンダー・リストがグレーアウト）
- 「確定を取り消す」で再編集可能に戻せる
- 月が変わると未確定状態にリセット（新しい月は常に未確定から始まる）

## 3. API仕様

Cloudflare Pages Functions（`functions/` ディレクトリ）で実装する。

### 3-1. 勤怠データ API

#### `GET /api/kintai`
指定ドクター・月の例外データを取得する。

| パラメータ | 型 | 説明 |
|-----------|-----|------|
| empId | string | 従業員ID（例：1030） |
| year | number | 対象年（例：2026） |
| month | number | 対象月（例：4） |

- KVキー：`kintai:{empId}:{yyyy-MM}`
- レスポンス例：`{ extraWorkDays: ["2026-04-09"], absentRecords: [{date: "2026-04-10", type: "有給"}], timeChanges: [] }`

#### `POST /api/kintai`
例外データをKVに保存する。

```json
{ "empId": "1030", "year": 2026, "month": 4, "data": { "extraWorkDays": [...], "absentRecords": [...], "timeChanges": [...] } }
```

フロントエンドからのデータ変更時、デバウンス1秒で自動POST。

### 3-2. 確定ステータス API

#### `GET /api/confirm`
指定ドクター・月の確定ステータスを取得する。パラメータは勤怠データAPIと同じ。
- KVキー：`confirmed:{empId}:{yyyy-MM}`

#### `POST /api/confirm`
確定ステータスを保存する。

```json
{ "empId": "1030", "year": 2026, "month": 4, "confirmed": true }
```

#### `DELETE /api/confirm`
確定ステータスを取り消す（再編集可能にする）。

### 3-3. CSV生成 API

#### `POST /api/generate-csv`
指定ドクター・月のCSVを生成して返却する。Cron Triggerとフロントエンドの両方から利用。  
CSVフォーマットはKING OF TIME打刻インポート仕様に準拠（Shift-JIS、ヘッダーコメント行付き）。

## 5. 月末自動CSV送信

### 5-1. 処理フロー

1. Cron Trigger が毎月最終日 20:00 JST に起動
2. `doctorList` の全ドクターについて KV から確定ステータスを確認
3. `confirmed: true` のドクターのみ処理対象
4. 各対象ドクターの例外データをKVから取得
5. フロントエンドと同じロジック（`lib/csv-generator.ts` に共通化）でCSVを生成
6. Shift-JISに変換
7. Resend API で山本さん宛にメール送信

### 5-2. メール仕様

| 項目 | 内容 |
|------|------|
| 送信先 | 山本さんのメールアドレス（環境変数 `RECIPIENT_EMAIL` に設定） |
| 送信元 | `kintai@star-dental.com` のような専用アドレス（Resendで設定するドメインに依存） |
| 件名 | 【勤怠データ】{ドクター名} {yyyy年M月}分（例：【勤怠データ】生野智也 2026年4月分） |

**本文テンプレート**

```
山本さん

{ドクター名}の{yyyy年M月}分の勤怠データをお送りします。

【勤怠サマリー】
  出勤日数：{workDays}日
  有給取得：{absentPaid}日（{有給日付リスト}）
  欠勤：{absentUnpaid}日
  振替休日：{absentSub}日
  休日出勤：{extraDays}日
  時間変更：{timeChangeCount}件

CSVファイルを添付しています。
KING OF TIMEへのインポートをお願いします。

※ このメールは勤怠管理アプリから自動送信されています。
```

**添付ファイル**
- ファイル名：`{yyyy}{MM}_{empId}_{empName}.csv`
- 文字コード：Shift-JIS
- フォーマット：KING OF TIME打刻インポートCSV仕様

### 5-3. 未確定ドクターへのリマインド

Cron実行時に未確定のドクターがいた場合、Slackの `#全体連絡` チャンネルに通知する。

```
【勤怠リマインド】4月分の勤怠データが未確定です：岡田先生、藤田先生。勤怠アプリで確定をお願いします。
```

### 5-4. Cron設定

`wrangler.toml` に以下を追加：

```toml
[triggers]
crons = ["0 11 L * *"]
```

※ UTC 11:00 = JST 20:00、`L` = 月の最終日
